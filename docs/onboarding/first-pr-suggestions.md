# First PR Suggestions

Low-risk tasks for first contribution. Pick one.

## Documentation

- Add term to glossary (`docs/glossary.md` — TBD)
- Improve convention example với clearer code
- Add JSDoc to public utility in `packages/utils/`
- Fix typo trong docs

## Code

- Add unit test for `cn` utility
- Add `Spinner` component to `@template/ui`
- Add new color token to `packages/ui/src/tokens/`

## Tooling

- Add useful npm script (e.g., `pnpm clean`)
- Improve verify script error messages

If unsure which to pick, ask Logan.

## Avoid for first PR

- Large refactors
- New ADRs (discuss first)
- Architectural changes
- Breaking changes
